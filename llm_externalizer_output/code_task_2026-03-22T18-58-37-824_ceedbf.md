### BUG: Catch block returns 400 for all errors
**Severity**: medium
**Location**: `POST` function, `catch` block
**Problem**: The `catch` block currently catches all exceptions and returns a generic "Invalid request body" error with a 400 status. This is problematic because errors other than an invalid request body (e.g., network issues, internal server errors during `buildPlugin` if not caught there, or unexpected errors during `request.json()`) will also result in a 400 status, which is misleading. A 400 status code specifically indicates a client-side error due to malformed syntax or invalid request parameters.
**Fix**: Differentiate between errors. Specifically, the `request.json()` call can throw an error if the request body is not valid JSON. This is the only scenario where "Invalid request body" and a 400 status is truly appropriate for the `catch` block. For other unexpected errors, a 500 Internal Server Error would be more suitable.

```typescript
export async function POST(request: NextRequest) {
  try {
    const body = await request.json();
    const result = await buildPlugin(body);

    if (result.error) {
      return NextResponse.json(
        { error: result.error },
        { status: result.status }
      );
    }
    return NextResponse.json(result.data, { status: result.status });
  } catch (error) {
    // Check if the error is due to invalid JSON parsing
    if (error instanceof SyntaxError && 'message' in error && error.message.includes('JSON')) {
      return NextResponse.json(
        { error: 'Invalid request body: Malformed JSON' },
        { status: 400 }
      );
    }
    // For any other unexpected errors, return a 500 Internal Server Error
    console.error("Unexpected error in POST /api/plugin-builder/build:", error); // Log the error for debugging
    return NextResponse.json(
      { error: 'Internal Server Error' },
      { status: 500 }
    );
  }
}
```
