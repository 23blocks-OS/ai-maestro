### BUG: Incomplete re-fetch on plugin toggle failure
**Severity**: medium
**Location**: `togglePlugin`
**Problem**: In `togglePlugin`, if the API call to toggle a plugin fails, `fetchPlugins()` is called to re-sync the UI. However, the `fetchElements()` call, which updates the list of active elements, is only made on successful API calls. If a plugin toggle fails, the plugin list will re-sync, but the elements list might remain out of sync, showing elements from a plugin that was supposed to be disabled (or vice-versa) but failed to update on the backend.
**Fix**: Call `fetchElements()` in the `finally` block of the `togglePlugin` function to ensure that the elements list is always refreshed after a plugin toggle attempt, regardless of success or failure. This guarantees consistency between the plugin state and the displayed elements.

```typescript

    setToggling(key)
    try {
      const res = await fetch('/api/settings/global-plugins', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ key, enabled: !currentEnabled }),
      })
      if (res.ok) {
        // Optimistic update
        setGroups(prev => prev.map(g => ({
          ...g,
          plugins: g.plugins.map(p => p.key === key ? { ...p, enabled: !currentEnabled } : p),
        })))
        setEnabledCount(prev => currentEnabled ? prev - 1 : prev + 1)
      } else {
        // If API call fails, re-fetch plugins to revert optimistic update
        fetchPlugins()
      }
    } catch (err) {
      console.error('Error toggling plugin:', err)
      // Re-fetch to resync UI with actual backend state after failure
      fetchPlugins()
    }
    finally {
      setToggling(null)
      // Always re-fetch elements to ensure consistency after a toggle attempt
      fetchElements()
    }
  }
```
