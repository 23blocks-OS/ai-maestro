**Problem**: The `activeOps` counter is decremented twice in certain error scenarios within the `buildPlugin` function. However, `runBuild`'s `.finally()` block will still execute and decrement `activeOps`. The issue is that the `runBuild`'s `.finally()` block is intended to handle the decrement for the *launched* build path. If an error occurs *before* `runBuild` is successfully launched (e.g., `fs.symlink` fails), `buildLaunched` would be `false`, and the outer `catch` would decrement.
