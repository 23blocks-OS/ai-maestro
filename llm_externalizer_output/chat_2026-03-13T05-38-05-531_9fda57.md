## 6. BLOCKED TASKS

### What Puts Task in Blocked State

**From `blocked-task-handling.md`:**

When agent reports blocker → Orchestrator verifies:
- [ ] Cannot self-resolve
- [ ] Not a knowledge gap
- [ ] Not a process issue
- [ ] Truly blocking (cannot continue work)

### Blocked Task Checklist

- [ ] Acknowledge blocker via AI Maestro
- [ ] Record current column BEFORE moving to Blocked
- [ ] Move task to Blocked column
- [ ] Remove current status:* label, add status:blocked
- [ ] Add blocker details as comment (include `Previous status: $CURRENT_STATUS`)
- [ ] Create separate GitHub issue for blocker (type:blocker label)
- [ ] Send blocker-escalation to AMAMA
- [ ] Check if other tasks can be assigned to waiting agent

### Unblock Flow

```
1. Verify blocker actually resolved
2. Retrieve previous status from blocker comment
3. Add resolution comment on blocked task
4. Close blocker issue: gh issue close $BLOCKER_ISSUE
5. Remove status:blocked label
6. Restore previous status label (e.g., status:in-progress)
7. Move task back to PREVIOUS column (not always In Progress)
8. Notify assigned agent via AI Maestro
9. Log resolution in issue timeline
```
