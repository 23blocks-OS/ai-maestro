### BUG: Type Mismatch in `params` Object
**Severity** / **Location** / **Problem** / **Fix**
**Severity**: Medium
**Location**: `GET`, `PUT`
**Problem**: The `params` object in the `GET` and `PUT` functions is typed as `Promise<{ id: string }>`. However, Next.js typically passes the `params` object directly, not as a Promise. This means `await params` will not behave as expected and might lead to runtime errors or incorrect `id` values if `params` is not a Promise.
**Fix**: Change the type of `params` to `{ id: string }` and remove the `await` keyword when destructuring `id`.
