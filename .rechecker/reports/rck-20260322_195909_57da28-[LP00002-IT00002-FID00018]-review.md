### BUG: Inconsistent error handling for `sendServiceResult`
**Severity**: medium
**Location**: `sendServiceResult` function
**Problem**: The `sendServiceResult` function checks for `result.error` to determine if an error response should be sent. However, it uses `result.status || 500` for the status code. If a service function returns a successful result with `status: 200` but also an `error` property (which shouldn't happen in a well-designed API but is possible), it will still be treated as an error. More importantly, if a service function returns an error object that *doesn't* have a `status` property, it defaults to `500`, which might not always be the most appropriate HTTP status code (e.g., a validation error might be 400). Conversely, if a successful result has no `status`, it defaults to `200`. This is generally fine, but the logic could be clearer.
**Fix**: Refine the `sendServiceResult` function to explicitly check for a successful status code range (e.g., 2xx) before assuming success, or ensure that service functions consistently return a `status` property with appropriate values for both success and error cases. For now, the most direct fix is to ensure that `result.error` implies an error status, and `result.data` implies a success status.

```typescript
// services/headless-router.ts
function sendServiceResult(res: ServerResponse, result: any) {
  if (result.error) {
    // If there's an error, use the provided status or default to 500
    sendJson(res, result.status || 500, { error: result.error }, result.headers)
  } else if (result.data !== undefined) {
    // If there's data and no error, assume success, use provided status or default to 200
    sendJson(res, result.status || 200, result.data, result.headers)
  } else {
    // Fallback for cases where neither error nor data is explicitly set,
    // or if a service returns an empty successful response.
    // This might indicate a problem in the service's return structure.
    // For now, assume 200 OK with an empty object if nothing else is provided.
    sendJson(res, result.status || 200, {}, result.headers)
  }
}
```
