### BUG: `parseMultipart` `latin1` Encoding Issue
**Severity**: high
**Location**: `parseMultipart`
**Problem**: The `parseMultipart` function converts the entire `body` Buffer to a `latin1` string (`body.toString('latin1')`) before splitting and processing. When extracting the file content, it then converts it back to a Buffer using `Buffer.from(content, 'latin1')`. This approach is problematic for binary files or text files with multi-byte characters (e.g., UTF-8). `latin1` is a single-byte encoding, and converting arbitrary binary data or UTF-8 text to `latin1` and back can lead to data corruption, as characters outside the `latin1` range will be misrepresented or lost.
**Fix**: The `parseMultipart` function should operate on the raw `Buffer` for content extraction, especially for file parts. Instead of converting the entire body to a string, it should find the byte offsets of the boundary and content headers, then extract the content as a sub-buffer. For text fields like 'options', it can then decode that specific sub-buffer using UTF-8.

```typescript
function parseMultipart(body: Buffer, contentType: string): { file: Buffer | null; options: string | null } {
  const boundaryMatch = contentType.match(/boundary=([^\s;]+)/)
  if (!boundaryMatch) return { file: null, options: null }

  const boundary = Buffer.from('--' + boundaryMatch[1])
  const crlf = Buffer.from('\r\n')
  const doubleCrlf = Buffer.from('\r\n\r\n')

  let file: Buffer | null = null
  let options: string | null = null

  let startIndex = 0
  while (startIndex < body.length) {
    const boundaryIndex = body.indexOf(boundary, startIndex)
    if (boundaryIndex === -1) break // No more boundaries

    let partStartIndex = boundaryIndex + boundary.length + crlf.length
    if (partStartIndex >= body.length) break

    const headerEndIndex = body.indexOf(doubleCrlf, partStartIndex)
    if (headerEndIndex === -1) break

    const headersBuffer = body.subarray(partStartIndex, headerEndIndex)
    const headers = headersBuffer.toString('utf8') // Headers are typically ASCII/UTF-8

    const contentStartIndex = headerEndIndex + doubleCrlf.length
    let contentEndIndex = body.indexOf(crlf, contentStartIndex)
    if (contentEndIndex === -1) { // Last part might not have trailing CRLF before next boundary or epilogue
      contentEndIndex = body.indexOf(boundary, contentStartIndex)
      if (contentEndIndex === -1) {
        contentEndIndex = body.length // Assume it's the very end
      } else {
        contentEndIndex -= crlf.length // Adjust for CRLF before boundary
      }
    }

    const contentBuffer = body.subarray(contentStartIndex, contentEndIndex)

    if (headers.includes('name="file"')) {
      file = contentBuffer
    } else if (headers.includes('name="options"')) {
      options = contentBuffer.toString('utf8') // Options are expected to be JSON string
    }

    startIndex = contentEndIndex
    if (body.indexOf(crlf, startIndex) === startIndex) { // Skip trailing CRLF if present
      startIndex += crlf.length
    }
  }

  return { file, options }
}
```
