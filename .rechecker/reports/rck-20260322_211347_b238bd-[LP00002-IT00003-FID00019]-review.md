### BUG: `findSkillsInDir` and `findScriptsInDir` do not handle symlinks securely
**Severity**: high
**Location**: `findSkillsInDir`, `findScriptsInDir`
**Problem**: In `findSkillsInDir`, the code explicitly skips symlinks when iterating through directory entries (`if (entry.isSymbolicLink()) continue`). However, the `fs.realpath` call at the beginning (`const realDir = await fs.realpath(dir)`) resolves the initial `dir` argument. If `dir` itself is a symlink, `realDir` will point to the symlink's target. If this target is outside the intended temporary scan directory, subsequent checks like `realPath.startsWith(realDir)` might still allow traversal outside the intended sandbox if `realDir` is already outside. While `scanRepo` creates a temporary directory and clones into it, ensuring `dir` is initially safe, the `fs.realpath` call could still be problematic if the temporary directory itself is a symlink (e.g., `/tmp` on some systems).

A more direct issue is that `findScriptsInDir` does *not* perform the same `fs.realpath` and path containment checks as `findSkillsInDir`. It only checks `entry.isSymbolicLink()` for files. If `scriptsDir` (which is `path.join(dir, 'scripts')`) contains a symlinked directory, `fs.readdir` on `scriptsDir` would follow it, potentially scanning outside the intended temporary directory.

**Fix**:
1. In `findSkillsInDir`, the `fs.realpath(dir)` should be used to establish a canonical root, and all subsequent paths should be resolved to their real paths and checked against this canonical root. The current check `if (realPath.startsWith(realDir))` is good, but it relies on `realDir` being the *intended* root. If `dir` itself is a symlink to an external location, `realDir` will be that external location. The `scanRepo` function creates `scanDir` in `os.tmpdir()`, which should be safe, but it's good practice to be defensive.
2. In `findScriptsInDir`, similar path containment checks should be added. Before calling `fs.readdir(scriptsDir)`, `scriptsDir` should be resolved to its real path, and this real path should be checked to ensure it's still within the original `dir`'s real path.
