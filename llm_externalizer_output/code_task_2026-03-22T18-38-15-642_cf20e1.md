### BUG: Inconsistent Error Handling for `scanRepo` Failures
**Severity**: medium
**Location**: `POST` function
**Problem**: The `POST` function handles errors returned by `scanRepo` by checking `result.error` and `result.status`. However, if `scanRepo` throws an exception (e.g., due to network issues, invalid URL format that bypasses initial validation, or internal service errors), this `try...catch` block will catch it. In such cases, the generic error message "Invalid request body" is returned with a 400 status, which is misleading. The error is not necessarily due to an invalid request body but rather an issue during the `scanRepo` operation.
**Fix**: Differentiate between errors caused by an invalid request body and errors originating from the `scanRepo` service. The `catch` block should provide a more general server error message or inspect the `error` object to provide a more specific message if possible.

```typescript
export async function POST(request: NextRequest) {
  try {
    const body = await request.json()

    if (!body.url || typeof body.url !== 'string') {
      return NextResponse.json(
        { error: 'Repository URL is required' },
        { status: 400 }
      )
    }

    const result = await scanRepo(body.url, body.ref || 'main')

    if (result.error) {
      return NextResponse.json(
        { error: result.error },
        { status: result.status }
      )
    }
    return NextResponse.json(result.data)
  } catch (error) {
    console.error('Error scanning repo:', error)
    // Check if the error is a JSON parsing error
    if (error instanceof SyntaxError && 'message' in error && error.message.includes('JSON')) {
      return NextResponse.json(
        { error: 'Invalid JSON in request body' },
        { status: 400 }
      );
    }
    // For other errors, assume it's an internal server error or an issue with scanRepo
    return NextResponse.json(
      { error: 'Failed to scan repository due to an internal server error.' },
      { status: 500 } // Use 500 for internal server errors
    )
  }
}
```
