**Severity**: high
**Location**: `act4_start_and_register`
**Problem**: The `safe_dir` variable is intended to escape characters that are special in `sed` replacement strings (`\`, `&`, `|`). However, the current escaping `sed 's/[\\&|]/\\&/g'` is incorrect. It escapes `\` and `&` by preceding them with a single `\`, which is correct for `sed`'s replacement string. But for `|`, it also precedes it with a single `\`. In `sed`'s replacement string, `|` is not a special character and does not need to be escaped. In `sed`'s replacement string, `&` is a special character that represents the entire matched pattern. So `\&` would insert a literal `&` preceded by the matched pattern, which is not the intended behavior. To insert a literal `&`, it needs to be escaped as `\\&`.
