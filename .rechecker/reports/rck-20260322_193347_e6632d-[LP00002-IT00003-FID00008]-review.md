### BUG: Inconsistent polling cleanup
**Severity**: medium
**Location**: `handleBuild`
**Problem**: The `handleBuild` function calls `clearPoll()` before starting a new `setInterval` for polling. However, if the initial build request (`fetch('/api/plugin-builder/build')`) fails or if `data.status` is not 'building', the `clearPoll()` function is not called again. This means that if a build is initiated, then fails immediately, and then another build is started, the `pollRef.current` might still hold a reference to a previous interval that was never cleared, leading to a potential memory leak or unexpected behavior if that interval tries to execute.
**Fix**: Ensure `clearPoll()` is called in all exit paths of `handleBuild` where a new poll is not explicitly started, or where the build process concludes without polling. Specifically, call `clearPoll()` after `setBuilding(false)` in the initial `if (!res.ok)` block, and in the `catch` block for the initial fetch. Also, call `clearPoll()` in the `else` block where `data.status` is not 'building'.
