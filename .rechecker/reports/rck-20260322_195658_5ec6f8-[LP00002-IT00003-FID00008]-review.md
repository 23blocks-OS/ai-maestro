### BUG: Incorrect `disabled` prop for "Push to GitHub" button
**Severity**: medium
**Location**: `BuildAction` component, JSX for "Push to GitHub" button
**Problem**: The "Push to GitHub" button's `disabled` prop is set to `!isComplete || pushing`. This means the button is disabled if the build is not complete OR if a push operation is currently in progress. However, the `showPush` state controls the visibility of the push section, which includes the input field for the fork URL and the actual push button. If `showPush` is `false`, the push section is hidden, but the "Push to GitHub" button itself is still enabled if `isComplete` is true and `pushing` is false. This allows users to click the button to toggle `showPush` even when `pushing` is true, which is not ideal UX. The button should be disabled if a push is already in progress, regardless of whether the push section is currently visible or not.
**Fix**: The `disabled` prop for the "Push to GitHub" button should only depend on `!isComplete`. The `pushing` state should disable the actual "Push" button inside the `showPush` section, not the toggle button itself.

```typescript
// Original:
<button
  onClick={() => setShowPush(!showPush)}
  disabled={!isComplete || pushing} // <-- This is the problematic line
  className="flex items-center gap-2 px-4 py-2.5 bg-gray-800 hover:bg-gray-700 disabled:bg-gray-800/50 disabled:text-gray-600 text-gray-300 font-medium rounded-lg border border-gray-700 transition-colors"
>

// Fix:
<button
  onClick={() => setShowPush(!showPush)}
  disabled={!isComplete} // <-- Remove '|| pushing'
  className="flex items-center gap-2 px-4 py-2.5 bg-gray-800 hover:bg-gray-700 disabled:bg-gray-800/50 disabled:text-gray-600 text-gray-300 font-medium rounded-lg border border-gray-700 transition-colors"
>
```
