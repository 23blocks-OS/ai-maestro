### BUG: `parseMultipart` uses `latin1` for content, potentially corrupting non-ASCII file data
**Severity**: high
**Location**: `parseMultipart`
**Problem**: The `parseMultipart` function converts the entire `body` buffer to a `latin1` string (`body.toString('latin1')`) before splitting and parsing. When it extracts the file content, it converts it back to a buffer using `Buffer.from(content, 'latin1')`. This approach is problematic for binary files or text files containing non-ASCII characters (e.g., UTF-8 encoded text). `latin1` is a single-byte encoding that maps byte values directly to characters. If the original file data contains multi-byte UTF-8 characters or arbitrary binary data, converting it to `latin1` string and back will corrupt the data, as `latin1` cannot represent all possible byte sequences correctly.

**Fix**: Instead of converting the entire body to a string, `parseMultipart` should work with buffers directly. The boundary splitting can be done on buffers, and the content extraction should avoid string conversions for the file part. For the `options` field, which is expected to be JSON, it's reasonable to convert that specific part to a UTF-8 string.

```typescript
// In parseMultipart function
function parseMultipart(body: Buffer, contentType: string): { file: Buffer | null; options: string | null } {
  const boundaryMatch = contentType.match(/boundary=([^\s;]+)/)
  if (!boundaryMatch) return { file: null, options: null }

  const boundary = Buffer.from('--' + boundaryMatch[1]) // Convert boundary to Buffer
  const parts: Buffer[] = []
  let currentPos = 0

  // Find parts by boundary
  while (currentPos < body.length) {
    const boundaryIndex = body.indexOf(boundary, currentPos)
    if (boundaryIndex === -1) break // No more boundaries

    // Extract the part between boundaries
    if (boundaryIndex > currentPos) {
      parts.push(body.slice(currentPos, boundaryIndex))
    }
    currentPos = boundaryIndex + boundary.length
  }

  // Remove preamble and epilogue (empty parts before first boundary and after last)
  const actualParts = parts.filter(part => part.length > 0 && !part.toString().trim().startsWith('--'))

  let file: Buffer | null = null
  let options: string | null = null

  for (const part of actualParts) {
    const headerEnd = part.indexOf(Buffer.from('\r\n\r\n'))
    if (headerEnd === -1) continue

    const headers = part.slice(0, headerEnd).toString('utf8') // Headers are usually ASCII/UTF-8
    let content = part.slice(headerEnd + 4)

    // Remove trailing \r\n if present
    if (content.length >= 2 && content[content.length - 2] === 0x0D && content[content.length - 1] === 0x0A) {
      content = content.slice(0, content.length - 2)
    }

    if (headers.includes('name="file"')) {
      file = content
    } else if (headers.includes('name="options"')) {
      options = content.toString('utf8') // Options are expected to be JSON string
    }
  }

  return { file, options }
}
```
