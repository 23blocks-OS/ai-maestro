### BUG: Inconsistent `portable_sed` usage for `INSTALL_DIR` escaping
**Severity**: medium
**Location**: `act4_start_and_register`
**Problem**: The `INSTALL_DIR` variable is escaped using `portable_sed` to create `safe_dir` for use in `CLAUDE.md` and `MAILMAN-CLAUDE.md` templates. However, the `portable_sed` function itself is called with `portable_sed "s|{{INSTALL_DIR}}|${safe_dir}|g" "$AGENT_DIR/CLAUDE.md"`. This means that `safe_dir` is already escaped, but `portable_sed` will escape the `|` character again if it's present in `safe_dir` because `portable_sed` uses `|` as a delimiter in its `sed` command. This double-escaping can lead to incorrect substitutions if `INSTALL_DIR` contains characters that `sed` interprets as metacharacters, especially `|`.
**Fix**: The `portable_sed` function should be modified to accept a delimiter as an argument, or a different approach should be used for escaping the `INSTALL_DIR` variable before passing it to `sed`. A simpler fix is to ensure that the `sed` command within `portable_sed` uses a delimiter that is guaranteed not to be in `safe_dir` after the initial escaping, or to perform the escaping of `INSTALL_DIR` directly within the `sed` command.

```bash
# Escape all sed metacharacters in INSTALL_DIR — used by both agent templates
# Escapes: \ & | [ ] . * ^ $ / (covers regex specials + our | delimiter)
local safe_dir
safe_dir=$(printf '%s' "$INSTALL_DIR" | sed 's/[[\.*^$|&\\\/]/\\&/g')

# Copy CLAUDE.md for first agent (only on fresh install, never overwrite)
if [ ! -f "$AGENT_DIR/CLAUDE.md" ] && [ -f "$INSTALL_DIR/scripts/FIRST-RUN-CLAUDE.md" ]; then
    cp "$INSTALL_DIR/scripts/FIRST-RUN-CLAUDE.md" "$AGENT_DIR/CLAUDE.md"
    # Substitute install-time variables (portable sed)
    portable_sed "s|{{INSTALL_DIR}}|${safe_dir}|g" "$AGENT_DIR/CLAUDE.md"
    portable_sed "s|{{VERSION}}|$VERSION|g" "$AGENT_DIR/CLAUDE.md"
    portable_sed "s|{{SELECTED_GATEWAYS}}|${SELECTED_GATEWAYS}|g" "$AGENT_DIR/CLAUDE.md"
fi
```
