### BUG: Missing dependency in `useEffect` for cleanup
**Severity**: high
**Location**: `useEffect` (empty dependencies)
**Problem**: The `useEffect` hook, which is responsible for cleaning up the polling interval on unmount, has an empty dependency array (`[]`). This means the `clearPoll` function it captures will be the one from the initial render. If `clearPoll` were to ever change (e.g., if it depended on state that changed), the `useEffect` would not re-run to capture the new version. While `clearPoll` is currently stable due to `useCallback` with empty deps, it's a common anti-pattern that can lead to stale closures if the `clearPoll` implementation changes in the future to depend on other state/props. More importantly, the `useEffect` itself should depend on `clearPoll` to ensure it always has the latest version.
**Fix**: Add `clearPoll` to the dependency array of the `useEffect` hook.
