More importantly, `trackH` should represent the height of the *track* itself, not the available space for the thumb to move. The actual scrollable range is `el.scrollHeight - el.clientHeight`. ### BUG: `positionOverlay` might hide overlay for legitimate scrollable content
**Severity**: low
**Location**: `positionOverlay` function
**Problem**: The condition `if (scrollHeight <= clientHeight + 1)` is used to determine if an overlay should be hidden. The `+ 1` is likely an attempt to account for floating point inaccuracies or minor differences. However, if `scrollHeight` is exactly equal to `clientHeight` (meaning no scrollbar is needed), or if it's just slightly larger (e.g., `clientHeight + 0.5`), the condition `scrollHeight <= clientHeight + 1` would still be true, and the overlay would be hidden. This is generally correct for truly non-scrollable content. The issue arises if `scrollHeight` is `clientHeight + 1`, meaning there is exactly 1 pixel of scrollable content. In this case, a scrollbar *is* technically needed, but the overlay would be hidden. While 1 pixel of scroll is rare, it's a logical inconsistency.

**Fix**: The condition should strictly check if `scrollHeight` is greater than `clientHeight` to determine if scrolling is possible. The `+ 1` threshold is arbitrary and can lead to edge cases where a very small scrollable area is incorrectly hidden.

```typescript
function positionOverlay(el: HTMLElement, overlay: HTMLDivElement, thumb: HTMLDivElement) {
  const { scrollTop, scrollHeight, clientHeight } = el
  // Change condition to strictly check if scrollHeight is greater than clientHeight
  if (scrollHeight <= clientHeight) { // No need for +1, if they are equal, no scrollbar needed.
    overlay.style.display = 'none'
    return
  }
  overlay.style.display = ''
  /* Position the overlay at the current scroll offset so it appears
     visually fixed at the top of the visible area */
  overlay.style.top = `${scrollTop}px`
  overlay.style.height = `${clientHeight}px`

  const ratio = clientHeight / scrollHeight
  const thumbH = Math.max(ratio * clientHeight, 40)
  const maxTop = clientHeight - thumbH
  // This condition `scrollHeight - clientHeight > 0` is correct for calculating `top`
  const top = scrollHeight - clientHeight > 0
    ? (scrollTop / (scrollHeight - clientHeight)) * maxTop
    : 0
  thumb.style.top = `${top}px`
  thumb.style.height = `${thumbH}px`
}
```
