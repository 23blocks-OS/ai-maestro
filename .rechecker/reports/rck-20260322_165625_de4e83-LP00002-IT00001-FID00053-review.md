### BUG: Incorrect GraphQL variable passing for arrays/objects in ghGraphQL
**Severity** / **Location** / `ghGraphQL` / **Problem** /
The `ghGraphQL` function attempts to pass arrays and objects as JSON strings using the `-F` flag. However, the `gh` CLI's `api graphql` command expects JSON variables to be passed as a single string with the `--raw-field` or `-f` flag, not as separate `-F` flags for each key-value pair. When using `-F`, `gh` interprets the value as a literal string, not as JSON. This will lead to GraphQL errors when complex variables are used.

**Fix** /
Modify the `ghGraphQL` function to pass JSON-stringified arrays and objects using the `-f` flag with the `key=value` format, where `value` is the JSON string.

```typescript
// /Users/emanuelesabetta/ai-maestro/.claude/worktrees/rck-de4e83/lib/github-project.ts
function ghGraphQL(query: string, variables: Record<string, unknown> = {}): unknown {
  const args = ['api', 'graphql']
  args.push('-f', `query=${query}`)
  for (const [key, val] of Object.entries(variables)) {
    if (typeof val === 'number' || typeof val === 'boolean') {
      args.push('-F', `${key}=${val}`)
    } else if (Array.isArray(val) || (typeof val === 'object' && val !== null)) {
      // Arrays and objects must be passed as JSON via -f so gh parses them correctly
      args.push('-f', `${key}=${JSON.stringify(val)}`) // Changed -F to -f
    } else {
      args.push('-f', `${key}=${String(val)}`)
    }
  }

  const result = execSync(`gh ${args.map(a => `'${a.replace(/'/g, "'\\''")}'`).join(' ')}`, {
    encoding: 'utf-8',
    timeout: 15000,
    stdio: ['pipe', 'pipe', 'pipe'],
  })

  return JSON.parse(result)
}
```
