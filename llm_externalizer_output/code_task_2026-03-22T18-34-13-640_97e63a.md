### BUG: Incorrect `getSkillKey` for Repo Skills
**Severity**: high
**Location**: `getSkillKey`
**Problem**: The `getSkillKey` function for `repo` type skills does not include the `ref` (branch/tag) in the generated key. This means if a user scans the same repository URL but with a different branch, the skills from the different branches will generate the same key, leading to potential conflicts or incorrect deduplication in `selectedKeys`. For example, `repo:https://github.com/user/repo.git:skill-path` would be the same for `main` and `dev` branches.
**Fix**: Include `skill.ref` in the key generation for `repo` type skills.

```typescript
// /Users/emanuelesabetta/ai-maestro/.claude/worktrees/rck-63bdc1/components/plugin-builder/SkillPicker.tsx
export function getSkillKey(skill: PluginSkillSelection): string {
  switch (skill.type) {
    case 'core':
      return `core:${skill.name}`
    case 'marketplace':
      return `marketplace:${skill.id}`
    case 'repo':
      return `repo:${skill.url}:${skill.ref}:${skill.skillPath}`
  }
}
```
