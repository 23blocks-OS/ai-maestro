### BUG: Inconsistent `getSkillKey` normalization for `repo` type
**Severity**: critical
**Location**: /Users/emanuelesabetta/ai-maestro/.claude/worktrees/rck-57da28/components/plugin-builder/SkillPicker.tsx::getSkillKey
**Problem**: The `getSkillKey` function for `repo` type skills uses `skill.ref || 'main'` to normalize the `ref` value. However, the `RepoScanner` component, specifically in `handleAddSkill` and when constructing the `key` for `selectedSkillKeys.has(key)`, normalizes the `ref` using `ref.trim() || 'main'`. If `skill.ref` is an empty string (e.g., `''`) in `SkillPicker.tsx`, `skill.ref || 'main'` will correctly resolve to `'main'`. But if `skill.ref` is a string containing only whitespace (e.g., `'   '`), `skill.ref || 'main'` will evaluate to `'   '` because a non-empty string is truthy. This creates an inconsistency where `getSkillKey` might produce a key with a whitespace-only `ref`, while `RepoScanner` will always produce a key with `'main'` for such cases due to `ref.trim()`. This leads to `selectedKeys.has(key)` failing to find a match for skills added from `RepoScanner` if the `ref` input was only whitespace.
**Fix**: Modify the `getSkillKey` function to trim the `skill.ref` before the `|| 'main'` fallback, ensuring consistent normalization with `RepoScanner`.
