### BUG: `MAX_BUILD_RESULTS` eviction logic can be inefficient and potentially evict too many
**Severity**: low
**Location**: `evictStaleBuildResults`
```typescript
  // If still over limit, evict oldest
  if (buildResults.size > MAX_BUILD_RESULTS) {
    const entries = [...buildResults.entries()]
      .sort((a, b) => new Date(a[1].createdAt).getTime() - new Date(b[1].createdAt).getTime())
    const toRemove = entries.slice(0, entries.length - MAX_BUILD_RESULTS)
    for (const [id] of toRemove) {
      buildResults.delete(id)
      const buildDir = path.join(BUILDS_DIR, id)
      fs.rm(buildDir, { recursive: true, force: true }).catch(() => {})
    }
  }
```
**Problem**: The logic to evict the oldest builds when `buildResults.size` exceeds `MAX_BUILD_RESULTS` is inefficient. It converts the `Map` to an array, sorts it, slices it, and then iterates to delete. Sorting the entire map every time this condition is met can be costly if `MAX_BUILD_RESULTS` is large or if the map frequently exceeds the limit. A more efficient approach for a fixed-size cache is often to use a data structure like `LRUCache` or simply maintain a queue of build IDs to easily identify the oldest. Additionally, `entries.length - MAX_BUILD_RESULTS` could result in `toRemove` containing more items than strictly necessary if `MAX_BUILD_RESULTS` is very small compared to the current size, although the primary issue is performance.

**Fix**: For a simple `Map`, a more efficient way to get the oldest `N` items without sorting the entire collection would be to iterate through the map and keep track of the `N` oldest. However, a simpler and more common pattern for this kind of eviction is to use a `Map` where insertion order is preserved (which JavaScript `Map`s do) and simply delete the `Map.keys().next().value` (the first/oldest entry) until the size constraint is met.

```typescript
  // If still over limit, evict oldest
  while (buildResults.size > MAX_BUILD_RESULTS) {
    const oldestId = buildResults.keys().next().value; // Get the first (oldest) key
    if (oldestId) {
      buildResults.delete(oldestId);
      const buildDir = path.join(BUILDS_DIR, oldestId);
      fs.rm(buildDir, { recursive: true, force: true }).catch(err => {
        console.warn(`Failed to remove oldest build directory ${oldestId} during eviction:`, err);
      });
    } else {
      // Should not happen if size > 0, but as a safeguard
      break;
    }
  }
```
This change makes the eviction process more efficient by only iterating and deleting the necessary number of oldest entries, rather than sorting the entire collection.
