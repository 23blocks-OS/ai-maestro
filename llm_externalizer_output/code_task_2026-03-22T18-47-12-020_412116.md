**Fix**: For a simple counter like this, the impact is likely minimal and the `Math.max` prevents negative values. However, for strict correctness in a concurrent environment, a more robust mechanism like a mutex or a queueing system would be used. Given the current design, the existing approach is probably acceptable for the intended scale, but it's a theoretical race condition.
