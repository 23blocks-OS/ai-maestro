### BUG: Inconsistent Plugin Name Validation
**Severity**: medium
**Location**: `PluginBuilderPage`
**Problem**: The validation for the plugin name in `isValid` and `disabledReason` is inconsistent.
The `isValid` check for the plugin name uses `name.trim().length > 0 && /^[a-zA-Z0-9][a-zA-Z0-9_-]*$/.test(name)`.
However, the `disabledReason` check for the plugin name uses `!name.trim()` for the empty check and then `!/^[a-zA-Z0-9][a-zA-Z0-9_-]*$/.test(name)` for the regex check.
The regex `^[a-zA-Z0-9][a-zA-Z0-9_-]*$` requires the name to start with an alphanumeric character. If the name is just `_my-plugin` or `-my-plugin`, it would fail the regex but `name.trim().length > 0` would pass. This means `isValid` would be `false` but `disabledReason` would incorrectly show "Enter a plugin name" if `name` starts with a non-alphanumeric character allowed by the regex (e.g., `_` or `-`) but not by the first character constraint.
**Fix**: The `disabledReason` logic should align with the `isValid` logic for the plugin name. Specifically, the `!name.trim()` check should be replaced with a check that also considers the starting character constraint of the regex.
