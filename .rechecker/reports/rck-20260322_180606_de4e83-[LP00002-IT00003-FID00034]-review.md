### BUG: `lockedCount` Calculation in `toggleModifier` is Flawed
**Severity**: medium
**Location**: `toggleModifier`
**Problem**: In the `toggleModifier` function, the `lockedCount` is calculated using the current state of *all* modifiers (`ctrlState`, `altState`, `shiftState`, `metaState`) from the component's state. However, when `toggleModifier` is called, the `state` parameter passed to it represents the *current* state of the specific modifier being toggled, but the `setState` call (e.g., `setCtrlState`) is asynchronous. This means that `lockedCount` might not reflect the *intended* new state of the modifier being toggled, leading to incorrect decisions about whether a modifier can be locked. For example, if `ctrlState` is 'once' and `altState` is 'locked', and the user double-taps 'ctrl', the `lockedCount` will be 1. If the user then double-taps 'shift', `lockedCount` will still be 1 (because `ctrlState` hasn't updated yet), allowing 'shift' to become 'locked', potentially exceeding the 2-locked-modifier limit if 'ctrl' also becomes 'locked'.

**Fix**: The `lockedCount` should be calculated based on the *potential* new state of the modifier being toggled. When checking `lockedCount < 2`, the count should include the modifier currently being toggled *if* it's about to become locked, and exclude it if it's about to become 'off'. A simpler approach is to count existing locked modifiers *excluding* the one currently being toggled, and then check if adding the current one would exceed the limit.

Modify the `toggleModifier` function as follows:
```typescript
  const toggleModifier = useCallback((
    state: ModifierState,
    setState: (s: ModifierState) => void,
    lastTapRef: React.MutableRefObject<number>
  ) => {
    const now = Date.now()
    const isDoubleTap = now - lastTapRef.current < 350
    lastTapRef.current = now

    // Count how many modifiers are currently locked, excluding the one being toggled.
    const allModifierStates = [
      { state: ctrlState, setState: setCtrlState },
      { state: altState, setState: setAltState },
      { state: shiftState, setState: setShiftState },
      { state: metaState, setState: setMetaState },
    ];
    const otherLockedCount = allModifierStates.filter(
      (mod) => mod.state === 'locked' && mod.setState !== setState
    ).length;

    if (state === 'off') {
      // Double-tap → locked (if under limit), single tap → one-shot
      if (isDoubleTap && otherLockedCount < 2) { // Check limit for new locked state
        setState('locked');
      } else {
        setState('once');
      }
    } else if (state === 'once' && isDoubleTap) {
      // Upgrade one-shot to locked if under limit
      setState(otherLockedCount < 2 ? 'locked' : 'once'); // Check limit for new locked state
    } else {
      setState('off');
    }
  }, [ctrlState, altState, shiftState, metaState]); // Dependencies are correct for reading current states
```
