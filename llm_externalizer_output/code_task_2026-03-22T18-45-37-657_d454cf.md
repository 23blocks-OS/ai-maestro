### BUG: Inconsistent Error Handling in `runBuild`
**Severity**: medium
**Location**: `runBuild`
**Problem**: The `runBuild` function catches errors from `execPromise` but then updates the `buildResults` map with a new object. If `buildResults.get(buildId)` returns `undefined` (e.g., if the build was evicted between the initial check and the error occurring), the `existing` variable would be `undefined`, leading to a runtime error when trying to spread `...existing`. While `existing` is checked at the beginning of the function, it's not guaranteed to be present when the `catch` block is executed.
**Fix**: Ensure `existing` is re-checked or handle the case where it might be `undefined` within the `catch` block. A safer approach would be to retrieve the latest state of the build result from the map before updating it in the `catch` block.

```typescript
    // ... (rest of success logic)

      // we can log this or decide not to update. For now, just log.
      console.warn(`Build result for ${buildId} not found during error handling.`);
    }
  }
```

### BUG: Potential Race Condition in `buildPlugin`'s `catch` Block
**Severity**: medium
**Location**: `buildPlugin`
**Problem**: In the `buildPlugin` function, the `catch` block for the `runBuild` promise attempts to update the build status if `r && r.status === 'building'`. However, `r` is assigned from `buildResults.get(buildId)` *before* the `runBuild` promise is awaited. If `runBuild` completes very quickly and updates the status to 'complete' or 'failed' before the `catch` block of the outer promise is entered (e.g., due to an error in the `runBuild` setup itself, not the `execPromise`), this check might incorrectly reset the status or overwrite a more accurate status.
**Fix**: The `runBuild` function itself is responsible for updating the status. The outer `catch` block in `buildPlugin` should primarily handle errors that occur *before* `runBuild` is successfully initiated (e.g., `fs.mkdir`, `fs.writeFile`, `fs.symlink` errors). The `runBuild` function's internal error handling is sufficient for errors during the actual build process. The `finally` block for `activeOps` decrement should be outside the `catch` block to ensure it always runs.

```typescript

  const buildId = randomUUID()
  const buildDir = path.join(BUILDS_DIR, buildId)

  try {
    // Evict stale builds before adding new ones
    evictStaleBuildResults()

    activeOps++ // Increment activeOps here, before any async operations that might fail

    // ... (create build directory, generate manifest, write manifest, copy script, symlink src)

    // Initialize build result
    const result: PluginBuildResult = {
      buildId,
      status: 'building',
      logs: [],
      manifest,
      createdAt: new Date().toISOString(),
    }
    buildResults.set(buildId, result)

    // Run build asynchronously
    runBuild(buildId, buildDir, manifest).catch(err => {
      console.error(`Build ${buildId} failed unexpectedly:`, err)
      // This catch block is for errors that prevent runBuild from setting a final status.
      // If runBuild itself fails to start or has an unhandled promise rejection,
      // ensure the status is marked as failed.
      const r = buildResults.get(buildId)
      if (r && r.status === 'building') {
        buildResults.set(buildId, {
          ...r,
          status: 'failed',
          logs: [...r.logs, err instanceof Error ? err.message : String(err)],
          outputPath: undefined,
          stats: undefined,
        })
      }
    }).finally(() => {
      // This finally block ensures activeOps is decremented regardless of runBuild's outcome
      activeOps = Math.max(0, activeOps - 1)
    })

    return { data: result, status: 202 }
  } catch (error) {
    // This catch block handles errors *before* runBuild is initiated,
    // or errors in the setup phase (mkdir, writeFile, symlink).
    // Decrement activeOps here as well, since it was incremented.
    activeOps = Math.max(0, activeOps - 1)
    console.error('Error starting plugin build:', error)
    return { error: 'Failed to start plugin build', status: 500 }
  }
}
```
