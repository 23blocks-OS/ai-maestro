This means the `ref` property is part of the `PluginSkillSelection` object but is not used in `getSkillKey` to form the unique identifier. If two skills come from the same URL and skillPath but different refs, they would generate the same key, leading to incorrect selection state management.
**Fix**: The `getSkillKey` function for `repo` type skills should include the `ref` property to ensure uniqueness.

```typescript
// In SkillPicker.tsx
export function getSkillKey(skill: PluginSkillSelection): string {
  switch (skill.type) {
    case 'core':
      return `core:${skill.name}`
    case 'marketplace':
      return `marketplace:${skill.id}`
    case 'repo':
      // Include ref in the key to ensure uniqueness for skills from different branches/tags
      return `repo:${skill.url}:${skill.ref}:${skill.skillPath}`
  }
}
```
